# 請求書入力・自動振分機能の追加実装

## 背景

建設会社（株式会社ネクスト・ワン）の工事台帳管理GASアプリに、**最も重要な核心機能**を追加する。

### 現状の業務課題
- 毎月60〜80社の仕入先から**紙の請求書**が届く
- 1枚の請求書に複数の工事現場（台帳番号）の明細が混在している
- 現在は台帳Excelを**1つずつ開いて金額を入力し、閉じて次を開く**作業を繰り返している
- 年間約250件の工事台帳があり、この振り分け作業に膨大な時間がかかっている

### このアプリで解決すること
紙の請求書を見ながら「帳番号」と「金額」を入力するだけで、複数の工事台帳に一括で経費を振り分ける。台帳を1つずつ開く作業をゼロにする。

## 追加実装する内容

### 1. 新規シート: `invoices`（請求書履歴）

処理済みの請求書を記録し、「今月60社のうち何社処理済みか」を把握するためのシート。

```
invoices: ID(PK), 仕入先, 請求年月, 請求総額, 明細件数, 登録日時
```

`setupSheets()` に追加すること。

### 2. 新規シート: `vendors`（仕入先マスタ）

仕入先名の表記揺れを防ぐためのマスタ。

```
vendors: 仕入先名(PK), よみがな, 作成日時
```

`setupSheets()` に追加すること。

### 3. サーバーサイド関数の追加（Code.js）

#### `getVendors()`
vendorsシートから全仕入先を返す。

#### `saveVendor(vendorData)`
仕入先を保存。

#### `getInvoices(yearMonth)`
指定月の請求書処理履歴を返す。yearMonth未指定なら全件。

#### `saveInvoiceWithExpenses(invoiceData, expenseRows)`
**最重要関数。** 1つの請求書から複数の台帳に経費を振り分ける。

```javascript
/**
 * 請求書を保存し、明細を各工事台帳の経費として一括登録する
 * @param {Object} invoiceData - {仕入先, 請求年月, 請求総額}
 * @param {Array} expenseRows - [{台帳番号, 金額, 備考}, ...]
 *   台帳番号が空の行（共通経費等）はスキップするか、特別な台帳番号で処理
 * @return {Object} {success: true, invoiceId, expenseCount} or {error}
 */
```

処理の流れ:
1. `invoices` シートに請求書ヘッダーを保存
2. `expenseRows` の各行を `expenses` シートに追加（月=請求年月の1日、仕入先=請求書の仕入先名）
3. 台帳番号が空の行はスキップし、スキップ件数を返却に含める
4. 全件追加後に成功レスポンスを返す

#### `lookupProject(projectId)`
台帳番号から案件名を即座に返す軽量関数（入力中のリアルタイムバリデーション用）。

```javascript
/**
 * @param {string} projectId - 台帳番号
 * @return {Object} {found: true, 案件名, 客先名} or {found: false}
 */
```

### 4. クライアントサイド: 請求書入力モーダル（app.html）

#### 4-1. 起動方法

ナビゲーションバーの既存ボタン群に追加:
```html
<button class="btn btn-sm btn-success fw-bold" onclick="openInvoiceEntryModal()">
  <i class="bi bi-receipt-cutoff me-1"></i>請求書入力
</button>
```

#### 4-2. モーダルUI仕様

大きめのモーダル（`modal-xl`）で、紙の請求書を横に置きながら入力する想定。

**ヘッダー部:**
```
仕入先:   [プルダウン（vendorsマスタから） + 新規入力可能なcombobox]
請求年月:  [YYYY-MM 形式の月選択、デフォルト=今月]
請求総額:  [数値入力]
```

**明細テーブル（動的行追加）:**
```
行# | 台帳番号(入力) | 工事名(自動表示) | 金額(入力) | 備考(任意) | 削除
 1  | [    ]         | -               | [       ]  | [      ]   | ×
 2  | [    ]         | -               | [       ]  | [      ]   | ×
 ...
[+ 行追加]
```

**フッター部（常時表示）:**
```
入力合計: ¥X,XXX,XXX  /  請求総額: ¥9,370,000  [差額: ¥0 ✓一致]
```

差額が0でない場合は赤字警告を表示。

**ボタン:**
```
[登録する]  [キャンセル]
```

#### 4-3. インタラクション詳細

**台帳番号入力時の自動補完:**
- 台帳番号セルからフォーカスが外れた時（`onblur`）、`lookupProject()` を呼ぶ
- 見つかったら → 工事名を自動表示（緑文字）
- 見つからなかったら → 「該当なし」を赤文字で表示
- 空の場合 → 「（帳番号なし）」をグレーで表示

**合計の自動突合:**
- 明細の金額が変更されるたびに入力合計を再計算
- 請求総額と比較し、差額を表示
- 一致したら緑のチェックマーク、不一致なら赤の警告

**行操作:**
- 「+ 行追加」で空行を追加（初期15行、不足時に追加）
- 各行の「×」ボタンで行削除
- Tabキーで 台帳番号 → 金額 → 備考 → 次行の台帳番号、と流れるように移動

**登録時:**
1. 入力合計と請求総額の一致を確認（不一致の場合はconfirmダイアログで警告）
2. 台帳番号が空の行は除外して登録（除外件数を通知）
3. `saveInvoiceWithExpenses()` を呼ぶ
4. 成功したら → トースト通知「XX件の経費を登録しました」+ モーダルを閉じる
5. 現在のビューをリフレッシュ

### 5. クライアントサイド: 請求書処理一覧ビュー（任意・優先度低）

ナビバーに「請求書」ビューを追加し、月ごとの処理済み請求書一覧を表示する。
これは核心機能ではないので、時間があれば実装する程度で良い。

## 実装上の注意事項

### 既存コードとの整合性
- 既存の命名規則・コードスタイルに従うこと
- サーバー関数は `Code.js` の末尾に追加セクションとして追記
- クライアントコードは `app.html` の `<script>` 内の末尾に追加
- CSSは `styles.html` に追記

### パフォーマンス
- `lookupProject()` はユーザーの入力中に繰り返し呼ばれるため、軽量にする（全シート読み込みではなく、projectsシートのみを最小限で読む）
- 初回モーダル表示時にprojectsの台帳番号→案件名マップをクライアントにキャッシュし、以降はクライアント側で補完する方が高速

### データ整合性
- 経費の「月」カラムには `YYYY-MM-DD` 形式（月初日）で格納する（例: 2026年2月 → `2026-02-01`）。これは既存データの規約。
- 仕入先名は `vendors` マスタの表記に統一する
